设计模式原则：
1.开放封闭原则
2.单一职责原则
3.依赖倒转原则
4.迪米特法则（最小知识原则）
5.接口隔离
6.合成/聚合复用原则
7.里氏替换法则

通过增加类的方式来应对变化，将变化量抽象成类，当变化发生时扩展类以满足变化。
